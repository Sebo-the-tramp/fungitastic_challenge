\paragraph{Completed segmentation slice: DINOv3 + MLP.}
\Cref{tab:mlp-results} retains the completed slice that already exists in the repo. The main pattern is consistent across both resolutions: \texttt{cls+masked} is the best configuration, outperforming plain \texttt{cls} by 1.82 points at 224 and 1.23 points at 448. Mask-aware pooling alone stays close to \texttt{cls} at 224 but falls behind at 448, while plain patch averaging is clearly weak. In the latest 224 rerun, the exploratory register variants also trail \texttt{cls+masked}, so the strongest current evidence still favors global context plus segmented local evidence.

\paragraph{Completed classifier slice: DINOv3 + 224 $k$-NN.}
The current classifier comparison is only partially populated, but the 224 $k$-NN sweep already preserves the same rough ranking. \Cref{tab:knn-results} shows that \texttt{cls+masked} again gives the best result, narrowly exceeding \texttt{cls}, while \texttt{patch} is far behind. This is useful because it suggests that the segmentation effect is not unique to the MLP head.
