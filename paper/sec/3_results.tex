\section{Results}
\label{sec:results}

\paragraph{Result flow.}
After the teaser comparisons, the body of the paper should answer three follow-up questions from the README and the planned fusion extension: which frozen backbone family is strongest, whether diversity matters more than quantity when combining backbones, and which lightweight classifier extracts the most from those features. The next blocks are placed in that order so the paper reads from top-line segmentation comparisons to broader backbone and fusion ablations, then to the classifier comparison, before zooming into the currently completed DINOv3 slice.

\paragraph{Completed segmentation slice: DINOv3 + MLP.}
\Cref{tab:mlp-results} retains the completed slice that already exists in the repo. The main pattern is consistent across both resolutions: \texttt{cls+masked} is the best configuration, outperforming plain \texttt{cls} by 1.82 points at 224 and 1.23 points at 448. Mask-aware pooling alone stays close to \texttt{cls} at 224 but falls behind at 448, while plain patch averaging is clearly weak. In the latest 224 rerun, the exploratory register variants also trail \texttt{cls+masked}, so the strongest current evidence still favors global context plus segmented local evidence.

\paragraph{Completed classifier slice: DINOv3 + 224 $k$-NN.}
The current classifier comparison is only partially populated, but the 224 $k$-NN sweep already preserves the same rough ranking. \Cref{tab:knn-results} shows that \texttt{cls+masked} again gives the best result, narrowly exceeding \texttt{cls}, while \texttt{patch} is far behind. This is useful because it suggests that the segmentation effect is not unique to the MLP head.

\paragraph{What is still missing from the final story.}
The README-level paper is larger than the completed slice. The 200-class oracle teaser is now exported, but the 2000-class SAM-3 teaser is still a placeholder, the cross-family backbone sweep has no numbers yet, the new multi-backbone fusion ablation is still a placeholder, and the classifier table still needs prototype and linear-probe results. The current paper therefore reports a real first result slice while making the missing pieces explicit instead of leaving the manuscript structurally narrow.
