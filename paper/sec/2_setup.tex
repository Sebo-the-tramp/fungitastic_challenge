\paragraph{Dataset and evaluation scopes.}
The README-level paper is organized around two scopes: a 200-class oracle-mask teaser and a 2000-class SAM-3 teaser. The currently completed closed-set slice in the repo uses extracted features for 215 classes. According to the saved run metadata, this slice contains 22,445 train/validation samples and 9,763 test samples. For the MLP runs, the stored \texttt{train} and \texttt{val} folders are merged and re-split into 19,077 training and 3,368 validation samples for early stopping. For $k$-NN, the full 22,445-sample train/validation pool is used as the retrieval bank.

\paragraph{Class-frequency skew.}
The completed closed-set slice is strongly long-tailed, which is why the paper should explicitly report the split-wise label distributions in \Cref{fig:class-distribution}. In the merged train/validation pool used for learning, class counts range from 3 to 1,164 samples per class, with mean 104 and median 44; the top 10 classes already account for 29.8\% of all examples. The held-out test split is similarly skewed, ranging from 1 to 641 samples per class, with mean 50 and median 21, and its top 10 classes cover 33.5\% of the samples. This matters because a single overall accuracy number otherwise hides whether gains come from broad improvement or from already well-represented classes.

\paragraph{Segmentation conditions.}
The paper structure in the README distinguishes three segmentation regimes: no segmentation, oracle segmentation, and SAM-3 segmentation. In descriptor terms, this corresponds to global image tokens, whole-image pooled patch tokens, and mask-aware pooled patch tokens where the pooling weights come from either ground-truth masks or predicted masks. The README also calls out register-token features as an additional descriptor to test alongside whole-image and masked pooling.

\paragraph{Backbone families.}
The top-line teaser plots are defined over the DINOv3 suite. The broader backbone ablation requested in the README spans CLIP, SigLIP, MAE, DINOv1, DINOv2, DINOv3, and InternViT. At the current repo state, the completed runs populate only the DINOv3 ViT-7B slice.

\paragraph{Multi-backbone fusion.}
The draft now reserves a dedicated block for testing whether diversity matters more than quantity when several frozen backbones are combined. For backbone $m$, let $x_m$ denote the descriptor and let $W_m x_m \in \mathbb{R}^d$ be its learned projection into a shared space. A lightweight gate predicts a scalar or branch weight $\alpha_m$, and the fused representation is
\[
z = \sum_{m=1}^{M} \alpha_m W_m x_m .
\]
This is the minimal version of the user-facing idea ``project everything to the same space and gate it with $\alpha$''. The corresponding ablation should compare single backbones, diverse pairs such as \texttt{DINO+SigLIP}, larger mixtures, and the full ensemble to test whether diverse combinations saturate most of the gain.

\paragraph{Classifier families.}
The classifier ablation requested in the README includes prototype, $k$-NN, linear probe, and MLP. The completed evidence currently covers MLP runs at 224 and 448, plus a 224 $k$-NN sweep. Prototype and linear-probe tables are still pending.

\paragraph{Completed DINOv3 feature variants.}
Within the completed DINOv3 slice, the current paper writes up the descriptors that already exist in the repo: \texttt{cls}, \texttt{patch}, \texttt{masked}, \texttt{cls+patch}, \texttt{cls+masked}, and exploratory 224-only \texttt{register} and \texttt{cls+register} rows from the latest rerun. These variants directly implement the README idea of comparing whole-image, segmented, and register-style descriptors.

\paragraph{Metric.}
The dashboard tracks top-1 accuracy, so that is the metric reported here. We now also report the split-wise class-frequency distributions because the completed slice is strongly imbalanced. Stratified analyses by class frequency, mask area, and hard species pairs remain part of the planned follow-up rather than the completed draft.
