\subsection{Dataset}
We evaluate our approach in the FungiTastic dataset \cite{Picek_2025_CVPR}, a fine-grained benchmark characterized by subtle differences between classes and a highly imbalanced, long-tailed distribution. We consider only the portion of the dataset comprising segmentation masks. The subset contains approximately 13k images with segmentation annotations (Fig.~\ref{fig:fungitastic}) spanning over 200 different classes. 

To ensure consistent evaluation, we restrict the label space to 194 classes present in both training and test splits. We report results in one-shot, few-shot, and low data regimes. Unless otherwise stated, the evaluation is performed on the full test set without rebalancing.

\subsection{Evaluation Protocol}

\paragraph{General Baseline + Oracle.} We prompt SAM3 with the string ``mushrooms" which constitutes our macro taxonomic prompt and retain the output by threholding at 0.5. This constitutes the general ability of the model to segment all kind of mushrooms with class agnostic knowledge. We then assume an oracle classification of pixels in order to compute the macro taxonomic upper bound (i.e. how good is SAM3 at segmenting mushrooms).

\paragraph{Fine-Grained Baseline + Oracle.} To produce fine-grained SAM segmentations we cannot assume a priori knowledge; therefore, we would need to forward each test image 194 times (i.e., one for each class). This results in a segmentation cost that grows linearly with the number of classes. To this end, we assume that an oracle provides exact taxonomic scientific names, and we use these as prompts for SAM3. We then retain the output by thresholding at 0.3. This baseline is useful to investigate the abilities of SAM3 when prompted with exact classes (ex. ``boletus edulis" or ``amanita muscaria"). This also acts as our upper bound. 

\paragraph{Our Method} Our baseline proposal starts from macro taxonomic segmentations and aim to assign class labels in a separate stage through training-free prototype based algorithm. Our primary backbone is DINOv3 \cite{simeoni2025dinov3}, used as a frozen feature extractor. Unless otherwise stated, we use the \texttt{[CLS]} token as the feature representation for DINOv3. Here, segmentation costs are constant with respect to the label space.

\paragraph{Metrics.}
We report top-1 and top-5 classification accuracy, as well as mean IoU for segmentation. Additionally, we introduce classification-aware IoU (CaIoU), defined as:
\[
\mathrm{CaIoU} = \mathrm{IoU}(\hat{M}, M) \cdot \mathbf{1}[\hat{y} = y],
\]
which measures segmentation quality conditioned on correct classification.
